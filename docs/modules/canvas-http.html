canvas.get_pages (self, download_bool, req, opt)
Paginated GET. Most REST interfaces use pagination to control sizes out return data. This requires iteration of multiple requests to return a full collection of information. Since this can be quite slow, this function has a built-in cache feature that stores the data to disk and if desired re-reads this cache instead of slowly requesting the data again. Where the cache is stored can be customised in the config file.

Parameters:

  • self
  • download_bool true | false | "ask"
  • req string URL to GET from
  • opt table of optional parameters

Returns:

    table of the REST result

Usage:

    canvas:get_pages(true,self.course_prefix.."assignments")
